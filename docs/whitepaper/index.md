## 概述

Lumtract 是一个基于 Rust + Next.js 的 DAG（有向无环图）可视化平台。它致力于将复杂的拓扑数据转化为直观、流畅的交互体验。

**本文档不是规范**。本文档是**规范运行后的一个快照**。若要理解 Lumtract 为什么这样设计，请查阅 `docs/` 目录下的推导内核。

## 目的档案摘要

**目的陈述**：
> Lumtract 用户在 Web 浏览器中，需要以可视化的方式理解复杂的 DAG 数据结构，以达成快速理解系统拓扑与排查故障的价值。

**关键权重分配**：

| 约束 | 权重 | 理由 |
|---|---|---|
| P-001 韦伯阈值 | 极高 | 拓扑结构复杂，需清晰层级区分 |
| P-002 空间频率 | 高 | 减少视觉噪点，提升长时间注视舒适度 |
| P-010 热区 ≥ 44px | 高 | 触控设备使用 |
| P-020 工作记忆 | 中 | 依赖空间记忆 > 数字记忆 |
| R-003 能耗 | 中 | 闲置时停止动画，交互时流畅 |

**完整目的档案**：参见 [docs/archive.md#40-项目目的档案](docs/archive.md)

## 组件库

Lumtract 的组件基于上述设计令牌构建。交互状态（hover、focus、active）遵循 [推导引擎 § 反馈权](docs/derivation.md#342-冲突裁决顺位) 协议。

### LumtractButton

- **基础样式**：主强调色背景，白色文字，`12px` 圆角
- **交互状态**：
  - `hover`: 亮度提升，轻微辉光
  - `active`: 亮度提升，涟漪动画扩散
  - `focus`: `2px` 外环，确保键盘导航可见性
- **尺寸变体**：
  - `sm`: 移动端需覆盖为 `min-h-[44px]`（[DEBT] 热区追踪）
  - `md` / `lg`: 桌面端标准尺寸

**设计依据**：遵循 [PHYS:P-005] 注意力切换舒适区（200–350ms）、[PHYS:P-010] 热区 ≥ 44px。

## ⚠️ 违背内核的旧表述（已废弃）

以下表述来自 Lumtract 项目早期探索阶段（v0.9.x），**已被 v10.0.0 内核判定为推导无效**。保留于此作为历史标记。

| 旧表述（❌ 已废弃） | 新内核立场（✅ 当前） | 违反的公理 |
|---|---|---|
| ~~"Lumtract 必须使用暗色模式。"~~ | 暗色是特定约束集（OLED + 暗环境）下的诚实解，不是必然。浅色模式在日光场景下同样是诚实解。 | 公理三：约束是现实 |
| ~~"涟漪层必须是水蓝色。"~~ | 水蓝是 `[GENE]` 品牌基因，不是 `[PHYS]` 物理约束。未来品牌升级可更改此色，无需修改内核。 | 公理三：基因冒充约束 |
| ~~"圆角必须统一为 8px。"~~ | 8px 是本项目基于 4px 网格和 P-002 约束的解。其他项目可根据不同目的权重选择 4px、12px 或 16px。 | 公理三：约定冒充约束 |
| ~~"微光是水之波光的本质特征。"~~ | 微光是特定 `[PURPOSE]`（暗室场景 + 深海隐喻）下的诚实解，不是本质。 | 公理三：目的冒充本质 |
