## 〇 · 定义

> **水之波光没有默认形态。**
>
> 本文档中的一切具体值——暗色背景、水蓝涟漪、圆角容器、微光流转——都是特定目的下的帕累托最优解，不是普适真理。约束链不输出"应该是什么样"，只输出"在这个目的下，这些方案诚实，那些方案冒犯"。
>
> **警惕文档自身的审美惯性。** 作者偏好暗色、圆角、微光——这些偏好必须接受目的验证，不能借约束链之名。
>
> ---
>
> **水之波光，是设计、内容与人三者相遇时，唯一的真实。**
>
> 设计是水。内容是光。人是观测者，亦是参与者。
>
> 波光不在水中，不在光中，不在眼中。波光在三者相遇的那一瞬。
>
> 好的设计，就是让这一瞬恰到好处地发生。
>
> **触境**，是人伸手触碰水面时，感受到的回应。水有温度，有阻尼，有涟漪。人以指尖阅读水的纹理，水以涟漪回应人的意图。
>
> **微光流转**，不是特效。它是环境的规律性呼吸——一种"活的静态"。它只由人的行为驱动——滚动时在边缘泛起，停下时便消散。存在，但不定睛看便无法感知。它不追求被注意，只追求"不被注意地存在"。波光不起无风之浪。水不独立呼吸，水只在被触碰时泛起涟漪。
>
> **毛玻璃是微光流转的拟物化、高能耗变体。** 两者服务于相同的认知需求（界面有生命感），但毛玻璃暗示不存在的物理材质（厚度、折射、景深），且需要实时模糊计算持续消耗 GPU。微光流转是毛玻璃的诚实替代——相同的心理效果，零物理本体假设，零闲置能耗。
>
> **设计是约束空间中的诚实推演。** 当所有约束被诚实列出——碳基的生理边界、硅基的渲染限制、内容的逻辑结构——设计的可行空间被压缩到极小。约束给出了硬边界，排除了错误答案。边界之内，存在一组帕累托最优解。具体选择哪一个，由内容逻辑、项目上下文和工程约定共同决定。不是"我觉得这样好"，而是"在这些约束下，这些方案都是诚实的，我选择这一个，并清楚为什么"。
>
> **约束不是清单，是链。**
>
> 约束链不是从光子开始的。约束链是从**目的**开始的。光（内容）为什么存在？因为人有目的——任务、需求、意图。目的是约束链的起点——它决定"为什么有这个内容"，从而决定"内容是什么"，从而启动整个约束链的推演。目的不同，可行空间不同。但不能因为目的不同，就违反碳基契约。目的是约束链的**方向**与**权重分配器**，不是约束链的**豁免**。
>
> **目的必须可验证。** 目的不是产品经理的主观意志，而是服务对象真实需求的映射。目的验证通过任务分析完成：人来到这个界面，要完成什么？完成这个任务的最小信息集是什么？如果目的陈述无法通过任务分析还原为具体用户行为，目的即无效，约束链无法启动。
>
> 从目的出发，光穿过渲染引擎、显示面板、眼球光学系统、视网膜感光细胞、视觉皮层，最终抵达认知。链上的每一环都是真实的物理边界。设计形态不是"选择"出来的，它是这条约束链上所有妥协的累积结果。
>
> 水之所以呈现某种形态，不是因为水"想"这样，是因为容器、重力、温度共同挤压。设计之所以呈现某种形态，是因为目的、渲染精度、显示色域、子像素排列、空间频率敏感度、预测误差最小化、语义强制映射、认知负荷共同挤压。
>
> 当链上的任何一环变化——新的显示技术、新的渲染协议、新的认知科学发现——设计的可行空间随之变化，诚实的推演自然指向新的形态。水之波光不为任何技术时代而写，它为约束链的诚实推演而写。
>
> 约束链实际无限延伸。目的之前还有人的生物动机、社会需求、文化语境；神经之后还有情绪状态、个人记忆。水之波光在认知系统处切断，因为再往后进入不可设计的领域。我们只负责目的到认知这一段。切断点是实用主义的，不是物理必然。承认边界，是另一种诚实。
>
> "水无常形"指水不预设形态，不是指水排斥圆润。水的形态由约束决定——表面张力约束下，水滴是圆的；容器约束下，水是方的。数字界面的圆角与否，不由隐喻决定，由空间频率敏感度、格式塔闭合性、语义边界、容器-内容同构等约束共同挤压。圆润不是"拟物"，是特定约束集下的诚实解。直角亦然。
>
> **推导有纪律。** 每一个设计值的诞生，都必须先回答：这是什么感官通道？它的生理约束是什么？它的设备约束是什么？目的是什么？是否遗漏了链上的任何一环？四问无法回答，推导即无效。约束陈述必须是科学上可验证的物理事实，模糊的类比或过度简化的生理模型会导致推导链在实例层断裂。
>
> **工程约定必须显式标注。** 约束边界与工程约定是两种完全不同的东西。约束边界是物理现实（如韦伯阈值），不可协商。工程约定是协作简化（如4px网格），可随项目上下文调整。项目基因是品牌选择（如水蓝色相），不受约束链管辖但必须在推导中被承认。目的裁决是权重分配，决定哪些约束在此场景下占主导。四者不可混淆。
>
> **颜色是涟漪的语言。** 色相四层：涟漪层严格单色相且只管辖当前交互状态，信号层因色觉障碍包容使用多色相双重编码，内容语义层由内容语义强制映射，内容结构层优先使用字体样式和背景差异。水本身无色——所有可见的颜色都是涟漪，不是水。同一时刻，同一元素只表达一个最高意图——由逻辑层裁决，CSS只消费确定的状态。
>
> **容器有冗余。** 水不溢杯。精确不等于脆弱。
>
> **虚空有确认。** 水面上没有船，不能只是一片水。零态不是空白，是对虚无的主动确认。
>
> **项目有名字。** Logo是项目的名字，是最直接、最无侵入感的项目标识。不需要额外的纹理或偏色——名字本身已是基因。
>
> 波光是相遇。触境是相知。微光是行为在水面的回响，是毛玻璃的诚实替代。推演是约束空间中的诚实行走。约束链是相遇之前的全部物理现实。目的是链的第一推动，也是链的权重分配器。

## 一 · 三大实体

水之波光的根基，由三个实体构成。三者缺一不可。

| 实体 | 角色 | 哲学含义 |
|---|---|---|
| **水** | 设计 | 透明的容器，退后的承载者。无色，故能映万物。无我，故能显万象。可触碰，有温度，有阻尼。水不独立呼吸——水只在被触碰时泛起涟漪。水不溢杯，容器有冗余。水面无舟时，水自证其空。**水本身没有颜色、没有形态、没有温度——这些属性都是目的与约束共同挤压的结果。** 水在冰中是固态，在壶中是液态，在雾中是气态。设计在文本阅读中是直角高对比，在视频播放中是圆角低干扰。水不坚持任何形态，水只坚持诚实。 |
| **光** | 内容 | 被表达的信息。光是一切视觉的源头。光因人的目的而存在——没有目的，就没有内容。光从目的出发，穿过渲染引擎、显示面板、眼球光学系统，最终抵达认知——链上的每一环都在塑造它最终被感知的形态。 |
| **人** | 观测者与参与者 | 碳基生物。有生理边界，有情感节奏，有认知极限。不仅观看，更伸手触碰。人抱有目的——任务、需求、意图——目的是约束链的起点与权重分配器。人的行为驱动水的涟漪——因动而动，停下即静。人不是约束链的终点，人是约束链的源头——设计服务于人，故链上的每一环都必须以人的生理与认知边界为最终裁决。 |

**核心关系**：
- 人抱有目的，目的决定内容——光因目的而存在。
- 水没有光，是虚空。
- 光没有水，是无处落脚的射线。
- 光与水相遇，没有人，是无人知晓的美。
- 人伸手触碰水面，涟漪从指尖荡开——波光是相遇，触境是相知。
- 人滚动页面，光在边缘流动。人停下，光消散——微光是水对行为的回应。
- 人看到Logo，知道这是哪里——名字是项目在水面上的倒影。
- 光穿过层层介质抵达人眼——约束链上的每一环都在挤压可行空间。设计是链上所有妥协的累积。**目的决定哪些妥协被接受。**

## 二 · 核心框架：目的权重分配器

> 设计质量不是数值，是维度。维度之间不可约化为单一标量。

v7.2.0 中的"核心公式"是伪数学——无量纲、无测量协议、运算结构缺乏约束链支持。v9.0.0 将其彻底删除，重构为**目的驱动的推导框架**。

### 2.1 推导元结构

```
目的（任务分析验证）
    ↓
分配权重：哪些约束在此目的下是硬边界？哪些可以妥协？
    ↓
可行空间被挤压出特定形态
    ↓
诚实选择（必须标注来源：[PHYS]/[ENG]/[GENE]/[PURPOSE]）
```

**约束链不是单向管道，是目的驱动的加权网。** 同一约束在不同目的下挤压出不同可行空间。

### 2.2 评估维度

| 维度 | 含义 | 评估标准 |
|---|---|---|
| **目的适配度** | 设计的具体形态是否与经过验证的目的对齐？ | 任务分析验证：这个界面的核心任务是信息获取、内容沉浸、还是操作执行？ |
| **内容清晰度** | 人能否立刻回答：这是什么？我在哪？我能做什么？ | 首次访问用户能否在3秒内定位核心任务入口。 |
| **感知完成度** | 反馈是否落在碳基感官的感知窗口内？涟漪链是否完整？ | 因果审 + 生命审：每个状态变化是否有可感知的反馈？反馈时序是否在100ms-600ms窗口内？ |
| **掌控深度** | 人的每一个主动行为，是否都能激起对应的涟漪？ | 触境审：行为与反馈的映射是否一一对应？ |
| **设计自我意识** | UI是否有自我表达？是否在抢内容的风头？ | 结构审：删除该元素后，可读性/聚焦度/边界感是否下降？ |
| **约束链边界冒犯** | 设计是否越过了约束链上任何一环的硬边界？ | 生理审 + 能耗审 + 约束链审：是否违反对比度、热区、闪烁阈值？ |
| **参与感缺失** | 人是否被降格为被动的"看客"？ | 触境审：界面是否可触碰、可划过、可留痕？ |
| **虚空焦虑** | 空状态是否让用户陷入"是没加载出来，还是真的没有？"的不确定性？ | 零态审：空状态是否主动自证？ |

**设计的目标**：正向维度最大化，负向维度最小化。当维度冲突时，按冲突裁决顺位裁断。**目的适配度是元维度**——若目的不适配，其他维度的优化可能是方向性错误。

### 2.3 目的权重矩阵示例

| 约束 | 文本阅读（邮件/文档） | 视频播放（YouTube） | 代码编辑器 |
|---|---|---|---|
| **韦伯阈值** | 权重极高。层级压缩不可接受。 | 权重降低。界面退后优先。 | 权重极高。语法高亮依赖精确色差。 |
| **空间频率** | 权重高。直角最大化信息密度。 | 权重低。圆角软化边界可接受。 | 权重高。等宽字体+直角是诚实解。 |
| **工作记忆** | 权重高。导航项需严格限制。 | 权重低。导航极简。 | 权重中。面板管理需可视化。 |
| **能耗预算** | 权重中。 | 权重高。视频本身高能耗，UI需极简。 | 权重中。 |
| **挤压结果** | 浅色/深灰背景、直角、高对比、无微光 | 纯黑背景、圆角、微光增益、界面消失 | 深色背景、直角、语义色相、无微光 |

## 三 · 约束链：从目的到认知

**约束链不是从光子开始的。约束链是从目的开始的。** 光（内容）为什么存在？因为人有目的——任务、需求、意图。目的是约束链的起点——它决定"为什么有这个内容"，从而决定"内容是什么"，从而启动整个约束链的推演。

### 3.0 目的层：约束链的第一推动与权重分配器

**目的验证三问**：
1. **任务还原**：这个界面服务于人的什么具体任务？任务能否被还原为可观察的行为序列？
2. **最小信息**：完成该任务所需的最小信息集是什么？任何超出最小信息集的内容都是负债。
3. **需求对齐**：目的陈述是否与真实用户研究或任务分析对齐？还是仅源于内部假设？

**目的权重分配原则**：
- 同一约束在不同目的下权重不同。
- 目的是约束链的**方向**，不是约束链的**豁免**。
- 目的不能使碳基契约失效（韦伯阈值、热区、对比度等在任何目的下都是硬边界），但目的可以**重新排列**其他约束的优先级。

**示例**：韦伯阈值在"文本阅读"目的下权重极高（层级压缩不可接受），在"视频播放"目的下权重降低（界面退后优先，纯黑可行）。但韦伯阈值本身从未消失——它只是从"必须表达丰富层级"变为"可以接受极简层级"。

从目的出发，光穿过渲染引擎、显示面板、眼球光学系统，最终抵达认知。链上的每一环都是真实的物理边界。设计形态不是"选择"出来的——它是这条链上所有妥协的累积结果。

### 3.1 硅基约束：渲染与显示

**光的第一道闸门是渲染引擎，第二道是显示面板。水不是自由流动的，它被每一道闸门塑形。**

#### 3.1.1 渲染约束

| 约束 | 对设计的必然要求 |
|---|---|
| **[PHYS] Alpha合成的非直觉叠加** | 多状态同时激活时，各状态的半透明背景在渲染层混合，产生非预期中间色——这是无因之色，违反因果不虚。因此状态必须互斥。互斥由逻辑层裁决，输出确定的`data-state`属性。CSS只消费确定状态。 |
| **[ENG] 子像素渲染精度** | 字号对齐4px网格，简化协作与心算。在Retina屏幕上非4px对齐同样清晰，但需评估特定字体的hinting表现。 |
| **[PHYS] 能耗预算** | 闲置时持续动画必须停止。交互流转搭车已有GPU活跃期，能耗可忽略。 |
| **[PHYS] JS/CSS能力边界** | 无JS环境：邮件客户端、RSS阅读器、搜索引擎爬虫、用户禁用JS、Tor高安全模式。SSR的初始HTML在水合前呈现静态快照，不依赖交互能力，水合后恢复JS能力。真正无JS环境→不假设交互流转可用。无JS环境→状态互斥**重协商**为CSS有限互斥。 |

#### 3.1.2 显示约束

| 约束 | 对设计的必然要求 |
|---|---|
| **[PHYS] 色域边界** | 任何色相必须在sRGB色域内安全，除非目的明确需要广色域且设备已知支持。 |
| **[PHYS] 刷新率与LTPO低功耗机制** | 持续动画阻止面板刷新率降至1Hz。闲置时微光流转必须停止，允许设备进入低功耗状态。 |
| **[PHYS] 像素排列** | Pentile OLED的子像素借用可能导致1px纯色线条引入非预期色相（红/绿彩边），违反因果不虚——非预期色相没有设计因。分割线最小宽度 ≥ 2px，或采用亮度差异（无色相）替代色相差异作为层级区分手段。 |
| **[PHYS] 亮度范围与HDR映射** | 背景色的选择必须在目的层被验证。**没有"默认暗黑模式"。**<br><br>**文本密集型任务**（邮件、文档、代码）：背景需要保留足够的亮度空间供层级表达（分割线、零态、悬停）。`#121212` 在 OLED 上诚实，`#F8F9FA` 在 LCD 日光下诚实。<br><br>**沉浸式内容**（视频、游戏、画廊）：界面应退后，纯黑 `#000000` 可与暗环境融合，让内容成为唯一光源。此时零态极少出现，视觉压缩可接受。<br><br>**[PURPOSE] 裁决**：选择背景色前，先回答——这个界面的核心任务是信息获取还是内容沉浸？零态出现的频率是多少？ |
| **[ENG] 容器-内容同构** | 如果显示设备的物理边界是圆角的（屏幕圆角、窗口圆角），屏幕内的独立容器（卡片、弹窗、面板）的圆角半径与设备圆角形成嵌套比例（通常为1/2至1/5）。这一定律的合法性来自格式塔"连续性"（视觉边界曲线的平滑延续）和"相似性"（共享曲线语言）。**嵌套比例本身是[ENG]工程约定，不是格式塔的必然推论。** 屏幕直角时，容器圆角需独立论证（空间频率阈值），不应盲目继承设备圆角。圆角半径的约束函数：`R ≥ 2px/像素密度`（子像素渲染精度下限），`R ≤ 元素最小边长/4`（格式塔闭合性上限），1x屏幕上R需使边缘锯齿降至感知阈值以下。 |
| **[PHYS] 环境融合** | 在暗环境或沉浸式内容场景中，显示面板与物理环境的边界最小化是合理目标。但环境融合的实现方式受目的约束：若零态高频出现，环境融合不能牺牲零态自证；若零态极少出现，纯黑的环境融合是可行路径。**[PURPOSE] 目的裁决硬件。** |

**裁决**：设计是否诚实面对了硅基约束？是否考虑了显示硬件的物理畸变、设备圆角的嵌套同构、暗环境下的环境融合？是否假设了不存在的技术能力？

### 3.2 碳基约束：感知与认知

**光穿过显示面板后，进入人的眼球。碳基约束是约束链的第三道和第四道闸门，也是最终裁决者——硅基约束可以随技术进步而变化，碳基约束不会。人不会进化出新的视网膜。**

#### 3.2.1 感知约束

**基石一：亮度阈值与空间频率阈值**

人眼对亮度差异的感知阈值约为1%-2%（韦伯定律，ΔI/I ≈ k）。主观感觉强度与刺激强度的对数成正比（费希纳定律，S = k·log(I)）。

同时，人眼视觉系统对中高频空间信息（约3-5 cycles/degree）的敏感度最高，极高频和极低频的敏感度均下降。在1x屏幕上，直角边缘的像素阶梯属于高频信息，容易跨越感知阈值形成视觉噪音。

| 约束 | 推演的边界 |
|---|---|
| 层级之间的视觉差异必须跨越韦伯阈值 | 层级数量有限。不能无限叠加。 |
| 设计的自我表达必须低于有意识注意阈值 | "无我"是感知阈值的必然推论。 |
| 深度暗示的强度下限 | 相邻层级亮度差异 ≥ 2%。 |
| 微光流转的幅度 | 基于浅色背景（`#F8F9FA`）计算，1%-3%亮度差异，刚好在阈值附近。**在纯黑背景（`#000000`）上，同等alpha值产生的亮度差异远低于韦伯阈值，视觉通道不可依赖。** |
| 间距冗余下限 | 差异 < 25%时无法感知分组。极限小屏间距不低于10px。 |
| 边缘锐度与空间频率 | 人眼对中高频空间信息最敏感。在1x屏幕上，直角边缘的像素阶梯容易跨越感知阈值。圆角配合抗锯齿渲染，将角点处的曲率变化分散到更多像素，减少单像素突变，同时通过提供连续的边界曲率促进格式塔闭合性。**直角在信息密度优先的场景下同样是诚实解。** |

**基石二：时间感知——感知窗口与心流边界**

- **[PHYS] 100ms**：因果感知下限。
- **[PHYS] 200-350ms**：注意力切换舒适区。
- **[PHYS] 400-600ms**：可注意但不打断心流的上限。
- **[PHYS] 1秒**：等待焦虑阈值。
- **[PHYS] 10-20分钟**：持续注意力周期。

| 约束 | 推演的边界 |
|---|---|
| 微交互反馈必须 > 100ms | 否则反馈不存在。 |
| 转场必须落在200-350ms | 短则切换不完整，长则焦虑。 |
| 引导动画 ≤ 600ms，每会话 ≤ 3次 | 超出则打断心流。 |
| 交互流转消散时间 | 停止滚动后300-500ms内消散。 |

**基石三：运动控制——菲茨定律与触摸生理**

菲茨定律：`MT = a + b × log₂(D/W + 1)`。手指指尖接触面积约44-56px。屏幕边缘是"无限大"目标。

| 约束 | 推演的边界 |
|---|---|
| 热区 ≥ 44px | 否则误触率飙升。 |
| 屏幕边缘豁免 | 垂直于边缘方向可降至32px。 |
| 高频操作就近排列 | 减少移动时间。 |
| 破坏性操作必须可撤销 | 误差必然存在。 |

**基石四：格式塔原则与视觉搜索效率**

接近性、相似性、连续性、闭合性——碳基视觉系统的硬件特性。这些原则解释了"元素如何被组织成组"。

**视觉搜索效率**：在已分组的复杂场景中，人眼需要显著性线索来快速定位目标。显著性可以通过对比度、面积、位置或局部色彩差异实现。但显著性线索必须服务于内容层级，不能成为装饰噪音——过度显著性导致预测误差飙升。

| 约束 | 推演的边界 |
|---|---|
| 卡片间距必须反映逻辑关系 | 接近性强制。 |
| 同一层级元素必须外观相似 | 相似性强制。 |
| 裁切必须可见 | 闭合性强制。 |
| 光区流向的下一步入口 | 连续性强制。 |
| 容器冗余与边界感 | 内容触及边缘→闭合性失效。 |
| 零态必须自证 | 空无一物→闭合性无法触发→焦虑。 |
| 显著性线索必须服务于内容层级 | 表头需要显著性线索帮助列定位，但底色差异 ≤ 2.5:1。斑马纹的过度显著性导致预测误差飙升。 |
| 容器-内容同构 | 屏幕圆角是显示设备的物理边界。屏幕内的独立容器的圆角与屏幕圆角形成嵌套比例，促进格式塔连续性和相似性。**嵌套比例是[ENG]工程约定。** |

**基石五：视觉注意力——聚光灯模型与变化盲视**

中央凹只占视野约1%。注意力一次只能聚焦一个对象。变化盲视：人不会自动注意到所有变化。

| 约束 | 推演的边界 |
|---|---|
| 同一时刻最多一个视觉焦点 | 多个焦点 = 注意力被撕扯。 |
| 反馈必须发生在焦点处 | 涟漪从触碰点荡开。 |
| 转场必须引导注意力 | 无方向的变化 = 注意力丢失。 |

**基石六：记忆边界——工作记忆与识别优于回忆**

工作记忆容量约4个组块（Cowan, 2001）。早期米勒研究提出7±2，后续研究将有效容量修正为约4。导航项≤7是工程保守约定，实际建议≤5以确保认知舒适。识别优于回忆。

| 约束 | 推演的边界 |
|---|---|
| 导航项 ≤ 7（保守约定），建议 ≤ 5 | 超过则分组或折叠。 |
| 多步流程必须可视化 | 不要求人回忆"做到哪了"。 |
| 瞬时锚点必须可视化 | 标记是可见涟漪，不是记忆。 |

**基石七：情绪时间——等待心理学**

焦虑拉长时间。确定性缩短时间。不确定性 + 等待 = 焦虑的平方。

| 约束 | 推演的边界 |
|---|---|
| 操作后必须立即确认 | 确认消除不确定性。 |
| 不确定的等待必须可视化 | 进度条 > 无进度条。 |
| 涟漪消散 = 安心 | 消散提供"正在完成"的确定性。 |
| 零态必须自证其空 | 空无一物 = 不确定是否加载 = 焦虑。 |
| 等待必须可视化 | 静止暗示完成 = 不确定性 = 焦虑。 |

#### 3.2.2 认知约束

| 约束 | 推演的边界 |
|---|---|
| **[PHYS] 工作记忆容量**（Cowan, ~4组块） | 导航项 ≤ 7（保守约定），建议 ≤ 5。多步骤流程全部可视化。瞬时锚点可见。 |
| **[PHYS] 识别优于回忆** | 图标+文字双重编码。状态色+图标双重编码。选项可见，不要求回忆。 |
| **[PHYS] 情绪时间感知** | 等待必须可视化。确定性消除焦虑。零态必须自证。骨架屏绝不静止暗示完成。 |
| **[PHYS] 预测加工** | 大脑不是被动接收视觉信息，而是主动建立预测模型。当输入与预测一致时，认知负荷极低（流畅感）。当输入与预测不一致时，产生"预测误差"，消耗工作记忆资源，引发疲劳感。一致性不是美学选择——它是认知能耗最小化的物理必然。秩序感是预测误差最小化的主观体验。 |
| **[PHYS] 显著性退级原则**（预测加工的推论） | 大脑对已处理/已访问/已读的信息自动降低注意力分配。保持高显著性会增加预测误差——认知系统预期"已处理=低优先级"，但视觉系统仍提供高显著性信号，这种冲突消耗工作记忆资源。已访问链接不属于涟漪层，属于内容层辅助信息，其视觉显著性应退级至辅助信息层级。 |
| **[PHYS] 空间记忆** | 人脑海马体对空间位置的存储容量远超工作记忆（数百个位置点）。界面利用空间记忆时，元素必须严格对齐（网格），否则空间编码失效。主屏幕图标网格的合法性不来自工作记忆，来自空间记忆与肌肉记忆的联合编码。图标网格的列数由屏幕宽度与热区尺寸共同挤压。 |

**裁决**：设计是否诚实面对了碳基约束？是否假设了人具有不存在的能力？

## 四 · 十二戒律

### 第一条 · 内容唯一

**光不应被水遮蔽。**

任何界面存在后，人必须能立即回答三问：这是什么？我在哪？我能做什么？

**推演自**：3.2.1基石一（感知阈值）、3.2.1基石五（单焦点原则）。

### 第二条 · 因果不虚

**波光不起无风之浪。**

任何视觉差异必须有因有果。因：人的行为或系统状态变化。果：可被碳基感官感知的反馈。禁止无因之果，禁止无果之因。

**推演自**：3.2.1基石二（因果感知下限100ms）、3.2.1基石七（确定性消除焦虑）。

### 第三条 · 层级有序

**光有明暗，波有大小。信息如是。**

信息架构的层级必须严格映射为视觉层级。视觉层级工具（由强到弱）：位置、面积、对比度、留白量。禁止用颜色作为唯一的层级区分手段。

**推演自**：3.2.1基石一（层级差异跨越韦伯阈值）、3.2.1基石四（格式塔接近性、相似性）。

### 第四条 · 涟漪的生命

**波光升起，扩散，消散，归于平静。人看到这完整的节奏，心生安定。**

每个元素必须完整经历四阶段：升起、扩散、消散、复原。涟漪是链，不是环。断链之处，便是认知摩擦的源头。

**状态互斥原则**：多重状态叠加时，Alpha禁止在渲染层混合。组件在逻辑层维护单一状态优先级栈，输出确定的`data-state`属性。CSS只消费确定状态。状态权重：聚焦 > 按下 > 悬停 > 默认。

**无JS环境**：在无法执行逻辑层裁决的渲染环境中，状态互斥采用CSS层有限互斥。这是该约束空间中的帕累托最优解，不是降级，是不同约束链下的不同诚实解。必须标记`TODO: [DEBT]`，JS可用时迁至逻辑层。

**触觉回应的三个实现**：

| 子项 | 定义 |
|---|---|
| **自动完形** | 内容被裁切时，系统带阻尼微调对齐，让内容完整呈现。 |
| **刚好确定** | 引导性元素的视觉强度 = 刚好能在200ms内被无意识感知。 |
| **瞬时锚点** | 人主动在当前位置留下轻量标记。不存储，不跨会话，用后即弃。 |

**推演自**：3.2.1基石二（四阶段符合因果链感知）、3.2.1基石三（自动完形弥补运动误差）、3.2.2认知约束（锚点是可见涟漪）、3.1.1渲染约束（Alpha合成非直觉性→互斥迁至逻辑层）、3.1.1渲染约束（无JS环境→CSS有限互斥路径）。

### 第五条 · 主权不容

**水的清澈，不容外染。**

所有外部UI库必须通过适配器封装。核心样式由设计令牌层统一管理。

| 许可证 | 状态 |
|---|---|
| MIT / BSD / ISC | ✅ 允许 |
| Apache 2.0 | ⚠️ 需标注专利风险 |
| GPL / AGPL | ❌ 禁止 |

### 第六条 · 零硬编码

**水无常形，道无常名。**

所有设计值必须通过设计令牌注入。源码中禁止裸色值、裸像素值、裸断点值。

### 第七条 · 最小有效能量

**涟漪不在力大，而在必达岸。**

| 反馈类型 | 感知窗口 |
|---|---|
| 微交互 | 100ms – 150ms |
| 转场 | 200ms – 350ms |
| 引导 | 400ms – 600ms，每会话最多3次 |

**反馈有效性原则**：以最小必要变化量达成可感知反馈。目标是最小有效变化量。

**推演自**：3.2.1基石二（感知窗口）。

### 第八条 · 无我

**水无色，故能映万物。**

UI的默认状态是无自我表达的。任何视觉元素的引入，必须证明其服务于内容的呈现。

**结构不是装饰。** 为内容提供边界、聚焦、呼吸感的元素，是水的表面张力——不为被看见，只为让内容更好地被看见。

**裁决四问**（删除该元素后）：
1. 可读性下降了吗？
2. 聚焦度下降了吗？
3. 边界感模糊了吗？
4. **这个形态（圆角/直角、暗色/浅色、微光/静默）是目的推导的结果，还是文档默认审美的惯性？**

一项下降→结构，保留。三项均无影响→负债，删除。第四问否定→审美惯性，删除。

**影归于无**：阴影的唯一合法用途，是表达内容在注意力层级中的深度。**拟真阴影在需要表达物理深度的场景（拖拽排序、窗口堆叠、3D空间导航）中是诚实解；在不需要物理深度暗示的场景中是负债。** 阴影色相继承环境色或强调色，不使用纯黑。

**推演自**：3.2.1基石一（设计自我意识必须低于注意阈值）、3.2.1基石四（格式塔）。

### 第九条 · 包容万物

**水面映天，不分贵贱。**

设计输出必须包容任何视口、任何输入方式、任何环境、任何能力的人。WCAG AA为底线。用户数据隐私不可逾越。

**水随器而形**：设计令牌系统必须具备原生环境响应能力。暗黑模式不是"更高级"的呈现，是水在暗处的特定形态；浅色模式是水在亮处的特定形态。**哪种形态被默认，由目的裁决，不由审美偏好裁决。**

**推演自**：约束链的普适性——链上的每一环都可能因设备、环境、能力而变化。包容不是美德，是对约束链多样性的诚实。

### 第十条 · 碳基契约

**人是血肉之躯。设计服务于人，故设计必须尊重约束链上碳基环节的全部边界。**

| 边界类型 | 原则 |
|---|---|
| **视觉边界** | 禁止 > 3Hz闪烁。动效可终止。对比度 ≥ 4.5:1。响应`prefers-reduced-motion`。 |
| **注意力边界** | 单焦点原则。长流程提供断点。不阻断式索取注意力。 |
| **运动控制边界** | 热区 ≥ 44px。屏幕边缘豁免：垂直于边缘可降至32px。高频操作就近。破坏性操作可撤销。 |
| **能耗边界**（硅基约束，碳基动机） | 交互流转仅在用户行为发生时触发。闲置时持续动画必须停止。毛玻璃的实时模糊计算是过度能耗，微光流转是诚实替代。 |
| **渲染环境边界**（硅基约束，碳基动机） | 无JS环境不假设状态互斥的逻辑层可用。邮件客户端不假设交互流转可用。 |
| **显示硬件边界**（硅基约束） | 显示硬件可能引入非预期色相偏移（如Pentile彩边）。单色相是设计意图，最终渲染结果需在目标显示约束下验证。当硬件强制引入非预期色相时，这不是设计选择了多色相——这是无因之果，违反因果不虚。 |

**终极裁决问**：如果人手湿着、在摇晃的车厢里、眼睛疲劳、只想快点做完然后离开——这个设计还站得住吗？

**推演自**：3.2.1和3.2.2全部碳基基石的完整映射。能耗、渲染环境与显示硬件边界是3.1硅基约束在碳基契约中的管辖延伸。

### 第十一条 · 空间之道

**水随器而形，光随波而序。**

**光区梯度**：

| 光区 | 数量限制 | 视觉权重 | 职责 |
|---|---|---|---|
| **主光区** | 仅一个 | 最高 | 当前页面存在的唯一理由。 |
| **辅光区** | 不超过两个 | 中等 | 支撑主光区的相关信息或操作。 |
| **泛光区** | 按需 | 最低 | 导航、上下文。不争抢注意力。 |
| **无光区** | — | 零 | 不相关内容。不存在。 |

**卡片三法则**：

| 法则 | 内容 |
|---|---|
| **内容呼吸法** | 卡片大小由内容的最小呼吸空间决定。能小则小。 |
| **关系距离法** | 卡片间距由逻辑关系决定。近即相关，远即独立。 |
| **流式优先法** | 卡片在可用空间内自然流动填满。不预设列数。 |

**容器冗余原则**：

| 冗余类型 | 原则 |
|---|---|
| **宽度冗余** | 容器最小宽度 = 内容最小呼吸宽度 + 两侧内边距。 |
| **高度冗余** | 容器最小高度 = 内容最小呼吸高度 + 上下内边距 + 一行文字高度。 |
| **间距冗余** | 极限小屏下间距可缩至最小安全间距（10px），但不允许低于。 |

**裁决**：将容器内容替换为最长可能文本后，内容是否触及边缘？

**光区流向**：人的来路，即是去路。下一步入口自然出现在注意力移动的方向上。不依赖箭头、按钮。

**水的流向**：自上而下，从左到右，从左上到右下。F型扫描的生理必然 + 内容逻辑的共同约束。

**裁切指示**：被画面裁切的内容，必须可见。**具体策略由目的裁决：**

| 目的 | 裁切指示策略 | 推导 |
|---|---|---|
| 极简阅读器、纯文本流 | 物理裁切足够 | 信息密度低，用户注意力在文字流，边缘裁切自然触发闭合性。遮罩是负债。 |
| 复杂仪表盘、多列数据、卡片网格 | 物理裁切 + 线性半透明遮罩 | 信息密度高，用户需要显著性线索定位边界，双重指示减少搜索时间。 |
| 沉浸式画廊、全屏内容 | 无裁切指示 | 内容应溢出边界，暗示无限空间。裁切指示破坏沉浸。 |

**遮罩规范**（当目的需要时）：
在裁切边缘，覆盖透明→环境背景色的线性渐变。方向与裁切方向一致。alpha：0→约0.10-0.15。**禁止使用黑色或灰色Alpha渐变**。暗黑模式下，遮罩终点色 = `oklch(from var(--bg-dark) calc(l + 0.08) c h)`，确保感知均匀的亮度提升。

**为什么是遮罩而非阴影**：拟真阴影暗示物理光源，引入不存在的本体。遮罩是信息的直接声明——"这部分内容存在，但当前不可聚焦。"

**零态布局**：当内容为空时，界面不呈现为虚空。零态必须自证其空。

**零态视觉优先原则**：
零态自证的首要通道是**视觉**。原因：
1. **带宽**：视觉的信息吞吐量远超触觉/听觉。
2. **并行性**：视觉可同时处理空间位置和语义内容。
3. **效率**：200ms 内完成确认，不中断任务流。

**纯黑背景（`#000000`）上的零态困境**：
在 `#000000` 上，任何像素的亮度差异必须跨越 ~1% 阈值才能被有意识感知。这意味着：
- 涟漪层 alpha 0.02-0.06 在纯黑上产生的亮度差异远低于韦伯阈值，**视觉通道不可依赖**。
- 零态自证、分割线、微光流转等依赖视觉层级的机制在纯黑上被物理压缩。

**诚实选择**：

| 目的 | 零态策略 | 推导 |
|---|---|---|
| 沉浸式内容（视频、游戏），零态极少 | 接受视觉压缩。零态使用刚好跨越韦伯阈值的最低亮度差异（如`#101010`，约2.5%差异）。**不引入触觉/听觉补丁来假装视觉通道仍然完整。** | 视觉压缩的成本被环境融合的价值覆盖。 |
| 信息密集型任务（邮件、文档、仪表盘），零态高频 | **放弃纯黑。** 背景回退至`#121212`，保留完整的视觉层级表达空间。 | 零态高频出现，视觉压缩不可接受。纯黑在此目的下不是诚实解。 |

**零态要素**：

| 要素 | 浅色/深色模式 | 纯黑模式（仅当目的允许时） |
|---|---|---|
| **涟漪信号** | 提示文本 + 极简图标。图标：水蓝alpha约0.12-0.15，强制`cursor: default`，不可聚焦。文本：辅助文字色（`--text-assist`），对比度 ≥ 3:1。 | 图标/文本使用`#101010`（约2.5%差异），处于阈值边缘。明确接受"纯黑零态视觉信号微弱"的物理事实。 |
| **位置** | 空区域中心。 | 空区域中心。 |
| **裁决** | 空状态是否可能被误解为"加载中"或"出错"？ | 纯黑模式下，是否因为目的层验证通过而接受视觉压缩？ |

**零态不是空白。零态是对虚无的主动确认。纯黑不是零态的敌人，但纯黑确实压缩了视觉通道的可行空间——诚实的解是目的裁决，不是补丁掩盖。**

**推演自**：3.2.1基石四（格式塔）、3.2.1基石五（单焦点原则）、3.2.1基石三（菲茨定律）、3.2.1基石一（间距冗余 + 韦伯阈值硬边界）、3.2.1基石七（虚空焦虑消除）、3.1.2显示约束（环境融合）、[PURPOSE]目的权重分配。

### 第十二条 · 触境

**人伸手触碰水面，水以涟漪回应。这是触境——人与设计的相知。**

**触境三原则**：可触碰、可划过、可留痕。

**触境的裁决**：人闭上眼睛，只凭指尖——能否感受到这个界面的回应？

**推演自**：3.2.1基石三（菲茨定律与触摸生理）、3.2.1基石二（链条连续性）、3.2.2认知约束（锚点是可见涟漪）。

## 五 · 冲突裁决：当约束发生碰撞

### 冲突裁决顺位

| 优先级 | 层级 | 包含内容 |
|---|---|---|
| **第一优先** | **生存权** | 碳基契约：对比度、热区、无闪烁、可访问性。颜色双重编码。WCAG AA。 |
| **第二优先** | **识读权** | 内容唯一、层级有序、排版之道。 |
| **第三优先** | **反馈权** | 因果不虚、涟漪的生命：反馈必须存在，涟漪链不可断裂。 |
| **第四优先** | **流转权** | 微光流转、动效细节、视觉质感。最先被降级。交互流转因能耗可忽略，一般不降级。 |

### 冲突裁决示例

**示例一**：设备宽度极窄（320px），标题字号按层级差推导为32px，但容器宽度不足以容纳，导致标题换行超过3行，破坏阅读节奏（基石四·连续性）。

**裁决**：约束A（字号层级差 ≥ 20%）与约束B（阅读节奏不破碎）同为识读权（第二优先）。选择对内容清晰度影响较小的方案——缩小标题至26px（层级差约16%，低于20%阈值）。标记`TODO: [DEBT]`：`320px视口下标题层级差不足20% | 原因: 容器宽度硬约束 | 演进: 设备宽度 > 375px时恢复标准层级差`。

**示例二**：OLED纯黑背景（`#000000`）可利用OLED子像素关闭特性降低能耗，同时与暗环境融合让内容成为唯一光源。

**裁决**：[PURPOSE] 目的验证是关键。若项目为沉浸式视频播放器，零态极少出现，纯黑是约束空间内的诚实解，零态使用`#101010`（约2.5%差异）确保跨越阈值。若项目为邮件客户端，零态高频出现，视觉压缩不可接受——背景必须回退至`#121212`。纯黑不是默认答案，也不是默认禁忌，它是**需要目的许可证**的选择。

**示例三**：圆角 vs 信息密度。

**裁决**：在代码编辑器目的下，直角最大化内容面积，减少无意义曲线引入的视觉噪音，圆角是负债。在移动端社交卡片目的下，圆角软化边界，促进格式塔闭合性，圆角是增益。**[PURPOSE] 目的裁决形态。**

**示例四**：微光流转 vs 心流。

**裁决**：在长文阅读、写作、深度工作目的下，边缘微光打断心流，增加预测误差，微光流转是负债，应静默。在信息浏览、社交feed目的下，滚动反馈确认行为被接收，微光流转是增益。**[PURPOSE] 目的裁决流转。**

### 冲突裁决流程

```
1. 识别冲突。
2. 查找顺位。
3. 高优胜出。
4. 标记负债：TODO: [DEBT] + 原因 + 演进路径。
5. 触底保护：触及第一优先即停止让步。
```

## 六 · 推导纪律

**推演的诚实不来自类比、经验或权威。它只来自约束链上每一环的清晰陈述。**

### 推导四问

| 问 | 内容 |
|---|---|
| **第一问：感官通道** | 空间/时间/颜色/触觉？ |
| **第二问：碳基约束** | 这个通道的感知与认知边界是什么？ |
| **第三问：硅基约束** | 这个通道的渲染与显示限制是什么？ |
| **第四问：约束链完整性** | 这个值是否遗漏了链上的任何一环？目的是否被诚实陈述并通过任务分析验证？ |

### 四来源标注系统（v9.0.0 核心机制）

任何设计值输出必须携带来源标签：

| 标签 | 含义 | 可协商性 |
|---|---|---|
| **[PHYS]** | 物理约束。来自认知科学、生理学、物理学的可验证边界。 | ❌ 不可协商 |
| **[ENG]** | 工程约定。协作简化、心算便利、历史遗产。 | ✅ 可随项目上下文调整 |
| **[GENE]** | 项目基因。品牌色、Logo、特定美学选择。 | ✅ 由项目所有者决定 |
| **[PURPOSE]** | 目的裁决。特定任务场景下的权重分配结果。 | ⚠️ 随目的变化而变化 |

**示例**：
- `背景色: #121212` → `[PHYS: OLED子像素关闭特性] + [PURPOSE: 文本密集型任务需保留层级空间]`
- `涟漪色: #2B8CBE` → `[GENE: Helix Ecosystem品牌色] + [PHYS: sRGB色域安全]`
- `字号: 16px` → `[PHYS: 标准阅读距离人眼分辨率] + [ENG: 浏览器默认基准]`
- `圆角: 8px` → `[ENG: 4px网格倍数] + [PURPOSE: 移动端卡片软化边界]`

### 物理约束与工程约定的区分

- **[PHYS] 约束给出硬边界**：排除错误答案。不可协商。
- **[ENG] 约定给出协作简化**：在约束边界内的便利选择。可随项目上下文调整。
- **[GENE] 基因给出身份标识**：品牌层面的选择，不借约束链之名。
- **[PURPOSE] 裁决给出方向与权重**：同一约束空间在不同目的下的不同挤压结果。

### 禁止的推导方式

| 违规方式 | 示例 | 错误类型 |
|---|---|---|
| **类比推导** | "间距用了斐波那契，字号也可以。" | 未区分感官通道。 |
| **权威推导** | "GitHub看着舒服，所以用20px。" | 从表象倒推。 |
| **经验推导** | "我做设计十年了，这个间距一看就对。" | 无法追溯约束。 |
| **流行推导** | "现在大家都在用超大标题。" | 流行不是约束。 |
| **链节遗漏** | "这个动画在MacBook上60fps流畅，所以没问题。" | 遗漏了低端设备的GPU约束和`prefers-reduced-motion`。 |
| **链节假设** | "用户一定有JS。" | 在邮件、SSR水合前、JS禁用场景中状态互斥失效。 |
| **模糊量化** | "边缘锐度与空间频率成正比。" | 约束陈述不精确。 |
| **约定冒充约束** | "容器圆角必须是屏幕圆角的1/2，因为这是格式塔必然。" | 嵌套比例是[ENG]，格式塔只要求边界感一致。 |
| **审美惯性推导** | "水之波光一直用暗色，所以这个项目也用暗色。" | 文档默认审美冒充目的裁决。 |
| **基因冒充约束** | "涟漪层必须是水蓝，因为单色相是物理约束。" | 单色相是[PHYS]，水蓝是[GENE]。 |

## 七 · 颜色之道：涟漪的梯度语言

### 色相四层

颜色在水之波光中分为四个层级，各有独立的色相策略与约束边界：

| 层级 | 色相策略 | 用途 | 约束来源 |
|---|---|---|---|
| **涟漪层** | 严格单色相，只管辖当前交互状态 | 交互状态：默认/悬停/按下/聚焦。分割线、引导暗示、环境微光。 | [PHYS] 单色相减少认知噪音，降低转换成本。时间边界：历史交互状态不属于涟漪层。 |
| **信号层** | 多色相（暖红/琥珀/冷绿/水蓝） | 系统状态：错误/警告/成功/信息。未读信息的数字标签。 | [PHYS] 色觉障碍包容（约8%男性色觉异常），必须双重编码。未读信息归属信号层·信息/中性——红色用于未读是语义污染。 |
| **内容语义层** | 多色相（由内容语义强制映射） | 代码语法高亮、数据可视化、图表类别区分。 | [PHYS] 语义强制：色相必须映射到内容的固有语义。 |
| **内容结构层** | 无色相优先 | 正文结构化元素：粗体、斜体、删除线、内联代码、引用块、已访问链接。 | [PHYS] 结构区分优先使用字体样式和背景微差异。新色相必须有内容因。 |

**涟漪层单色相的硬件畸变免责声明**：涟漪层的单色相是设计意图。显示硬件（Pentile排列、色域映射、HDR色调映射）可能引入非预期色相偏移。设计必须在目标显示约束下验证最终渲染结果。当硬件强制引入非预期色相时，这不是设计选择了多色相——这是无因之果，违反因果不虚。

### 涟漪层的色相：项目基因，不是物理约束

> **涟漪层必须单色相（[PHYS] 认知噪音），但具体是哪个色相由项目基因（[GENE]）决定。**

水蓝 `#2B8CBE` 是 Helix Ecosystem 的项目基因，不是约束链的必然输出。约束只规定"涟漪层单色相"，不规定"水蓝"。若项目基因是琥珀色，涟漪层应使用琥珀梯度。若项目基因是中性灰，涟漪层应使用灰度梯度。

**v7.x-v8.0.0 的错误**：将水蓝写进了约束链，仿佛它是物理必然。这是 [GENE] 冒充 [PHYS]。

### 内容层色相策略

内容层不是无色的。代码语法高亮、数据可视化、图表类别等由内容语义强制要求的多色相表达，属于**内容语义层**，不受涟漪层单色相约束。

**内容语义层约束边界**：
1. **语义强制**：色相必须映射到内容的固有语义，不能随意指定。
2. **双重编码**：任何语义色相必须同时通过字体样式或形状传递。
3. **韦伯跨越**：相邻语义类型的亮度差异 ≥ 2%。
4. **色域安全**：所有语义色相在sRGB内安全，暗色模式下向高亮度偏移保持可辨识。

**正文结构化色相**（内容结构层）：

| 元素 | 归属 | 色相策略 | 推导 |
|---|---|---|---|
| **粗体** | 内容结构层·强调 | 字重增加，不加色相 | [PHYS] 变量叠加：字重优先于颜色。 |
| **斜体** | 内容结构层·语气 | 字形变化，不加色相 | [PHYS] 语气变化是同一层级的微差。 |
| **删除线** | 内容结构层·废弃 | 降低对比度（辅助文字色），不用删除线 | [PHYS] 删除线破坏闭合性。 |
| **内联代码** | 内容结构层·结构 | 等宽字体 + 背景微差异（≤ 2.5:1对比度），不引入新色相 | [PHYS] 结构区分，不是语义区分。 |
| **引用块** | 内容结构层·引用 | 左侧水蓝alpha 0.06边框 + 背景微差异 | [GENE] 水蓝为Helix品牌色 + [PHYS] 结构分割。 |
| **超链接（正文内）** | 涟漪层 | 水蓝alpha 0.15 + 下划线（悬停时） | [GENE] 水蓝 + [PHYS] 链接是交互元素，归属涟漪层。 |
| **已访问链接** | 内容层·辅助信息 | 辅助文字色（`--text-assist`）+ 虚线下划线 | [PHYS] 显著性退级原则。 |

### 信号层：未读信息

未读信息（Badge）归属**信号层·信息/中性**。它不是错误，不是警告——它是"有待处理的内容"。红色用于未读是信号层语义污染——把"信息"标记为"危险"。

**水之波光的替代方案**：

| 方案 | 能量 | 推导 |
|---|---|---|
| **数字标签**（水蓝alpha 0.40） | 低 | [GENE] 水蓝 + [PHYS] 信号层：信息/中性。数字本身提供精确信息。 |
| **引导流转** | 极低 | [PURPOSE] 用户滑动时，未读应用图标被微光掠过。 |
| **时间衰减** | 低 | [ENG] 未读的水蓝alpha随时间降低（24小时后降至0.15），表达"时效性衰减"。**24小时阈值是工程约定，基于日常任务周期假设，非认知科学必然推论。** |

### 水的涟漪色：水蓝 #2B8CBE（LCH: 50, 40, 260）

水蓝不是水的颜色，是**Helix Ecosystem 的项目基因**。水本身无色——所有可见的颜色都是涟漪，不是水。

### 涟漪梯度：一个色相，无数透明度

| 状态 | 透明度 | 权重 | 背景验证 |
|---|---|---|---|
| **强调/聚焦** | alpha 0.40 | 最高 | 浅色/深色/纯黑均跨越韦伯阈值 |
| **选中/按下** | alpha 0.30 | 高 | 浅色/深色/纯黑均跨越韦伯阈值 |
| **活跃/悬停** | alpha 0.15 | 中 | 浅色/深色跨越；纯黑需验证 |
| **分割/引导** | alpha 0.06 | 低 | 浅色/深色跨越；纯黑不跨越 |
| **环境/微光** | alpha 0.02-0.04 | 最低 | 仅浅色/深色跨越；纯黑视觉通道失效 |

**纯黑背景（`#000000`）下的涟漪层降级**：当视觉信号低于韦伯阈值时，涟漪层不假装存在。交互流转在纯黑模式下仍可在边缘发生（搭车GPU活跃期），但引导流转和微光流转的视觉显现停止。**不自动激活触觉通道替代**——触觉是不同通道，不能替代视觉完成层级定位。若视觉压缩不可接受，回退背景色。

### 状态互斥与逻辑层裁决

多重状态叠加时，逻辑层维护单一状态优先级栈，输出`data-state`。CSS只消费确定状态。优先级：focus > active > hover > default。

### 状态色：双重编码

| 状态 | 颜色 | 辅助编码（至少一项） |
|---|---|---|
| **危险/错误** | 暖红（偏橙） | 图标 + 文字 |
| **警惕/警告** | 琥珀/暖黄 | 图标 + 文字 |
| **成功/通过** | 冷绿（偏蓝绿） | 图标 + 文字 |
| **信息/中性** | 水蓝 | 图标 + 文字 |

### 对比度：4.5:1下限 + 10:1上限

| 层级 | 最小 | 最大 |
|---|---|---|
| **正文** | 4.5:1 | 约10:1 |
| **辅助文字** | 3:1 | 约7:1 |
| **背景与卡片** | 约1.5:1 | 约2.5:1 |

浅色模式：正文 `#1A1A1A`，背景 `#F8F9FA`。暗黑模式：正文 `#E0E0E0`，背景 `#121212`。OLED纯黑（`#000000`）与零态自证不冲突，但需 [PURPOSE] 目的许可证。

## 八 · 微光流转：水之波光的指纹

**微光流转是"活的静态"。它只由人的行为驱动。波光不起无风之浪。**

它服务于两个目标：**灵动**（界面有生命感）与**反馈确认**（行为被接收的感知）。

**它不追求被注意，只追求"不被注意地存在"。**

**微光流转不是默认配置。** 其存在与否由目的裁决。

| 目的 | 微光流转 | 推导 |
|---|---|---|
| 长文阅读、写作、深度工作 | ❌ 负债 | [PHYS] 边缘微光打断心流，增加预测误差。用户需要"界面不存在"的沉浸感。 |
| 信息浏览、社交 feed、仪表盘 | ✅ 增益 | [PHYS] 滚动反馈确认行为被接收，边缘微光暗示内容边界。 |
| 视频播放、游戏 | ⚪ 中性 | [PHYS] 滚动行为极少，微光流转无触发因。不增加也不减少负债。 |

**v7.x-v8.0.0 的错误**：声称微光流转有益于"睫状肌调节/健康"，该声明缺乏强证据支持，属于"结果先行"的生理学背书。v9.0.0 予以删除。微光流转的合法性仅限认知与反馈层面。

### 毛玻璃：微光流转的拟物化变体（对比注释）

毛玻璃（如iOS Vibrancy）和微光流转共享同一个认知根源——它们都是"环境有生命"的暗示。但它们的实现路径完全不同：

| 维度 | 毛玻璃（iOS Vibrancy） | 微光流转（水之波光） |
|---|---|---|
| **本体论** | 拟真物理材质（厚度、折射、景深） | 无本体，只是光的流动 |
| **能耗** | 高（实时模糊计算，每帧GPU合成） | 极低（CSS渐变或简单着色器） |
| **信息内容** | "这是毛玻璃"（材质信息） | "此处有边界"（结构信息） |
| **因果** | 无因之果（材质不因用户行为变化） | 因动而动（滚动/触碰驱动） |

**裁决**：毛玻璃是微光流转的拟物化、高能耗变体。微光流转是毛玻璃的诚实替代——相同的心理效果（灵动、生命感），零物理本体假设，零闲置能耗。

### 流转的两种形态

**一、交互流转——水回应用户的行为**

用户滚动页面时，边缘出现极微弱的波光，沿滚动方向流动。停止后300-500ms内消散。幅度1%-3%亮度差异（基于浅色/深色背景计算）。**在纯黑背景上，交互流转的视觉幅度可能低于韦伯阈值，此时不触发视觉显现。** 能耗：搭车GPU活跃期，闲置时零能耗。一般不降级。

**二、引导流转——光掠过下一步的入口**

引导流转没有独立的驱动机制。它是交互流转路径与引导元素相交时的局部显现——因是滚动行为，果是引导元素表面的光带掠过。引导流转不可脱离交互流转单独存在。

一条极淡的光带（2-4px高）沿元素表面滑过。持续约400-600ms。刚好可感知，不打断阅读。

| 属性 | 传统引导 | 引导流转 |
|---|---|---|
| **存在方式** | 独立存在，需设计、维护、触发 | 交互流转路径上的天然节点 |
| **成本** | 每增一个，负债增加 | 不增加负债 |
| **打扰度** | 可能打断心流 | 低于打断阈值 |
| **驱动** | 独立（时间或事件） | 依附于交互流转（因动而动） |

**覆盖边界**：交互流转的触发行为限于由用户直接行为引起的视口变化：
- 滚动（鼠标滚轮、触摸滑动、拖拽）
- 窗口缩放
- 键盘焦点导航导致的自动滚动

**以下情况不触发交互流转/引导流转**：
- 程序化锚点跳转（非用户直接行为驱动）
- 页面初始加载完成后的自动定位
- 无需滚动即可完整呈现内容的页面（因果不虚：无滚动行为，无流转之因）

**能耗裁决**：

| 形态 | 触发条件 | 闲置能耗 | 降级优先级 |
|---|---|---|---|
| **交互流转** | 用户滚动 | 零 | 一般不降级。纯黑模式下视觉幅度不足时不显现。 |
| **引导流转** | 交互流转路径与引导元素相交 | 零（随交互流转触发） | 随交互流转同步。 |

**推演自**：3.2.1基石一（幅度在韦伯阈值附近，纯黑模式不显现）、3.2.1基石二（停止后300-500ms消散，引导流转400-600ms）、3.1.1渲染约束（能耗预算）、第二条戒律（因果不虚——引导流转无独立驱动）、[PURPOSE] 目的裁决（非默认配置）。

## 九 · 排版之道

**字的排列，是光在水面上的波纹。有序，则光被聚焦。无序，则光被散射。**

**空间与时间分治**：间距服务于格式塔（接近性→分组）。字号服务于眼动流（对焦→识别）。两者不可互换。

**间距系统的工程约定**：

间距服务于格式塔接近性（组间间距显著大于组内间距）。斐波那契数列被用作间距系统的工程约定，因其增长比率（≈1.618）能自然形成可区分的层级差异。**[ENG] 这不是认知科学的必然推论，是工程协作的便利选择。** 任何能确保组间/组内差异显著大于韦伯阈值的间距系统都是约束空间内的合法解。

### 正文节奏

正文不是均质的文字流。段落间距、引用块、内联代码、列表标记共同构成阅读的节奏。节奏由格式塔接近性（间距决定"哪些文字是一组"）和视觉搜索效率（显著性线索帮助定位结构元素）共同约束。任何打破节奏的元素——过度的色相跳跃、不一致的间距、突兀的字体变化——必须证明其服务于内容清晰度，否则是负债。

### 层级变量的优先级

| 优先级 | 变量 | 用途 |
|---|---|---|
| **1** | 空间位置 | 最高层级 |
| **2** | 面积/字号 | 标题 > 副标题 > 正文 |
| **3** | 字重 | 强调关键词 |
| **4** | 对比度 | 正文 > 辅助 > 禁用 |
| **5** | 间距 | 标题上下留白 > 段落间距 |
| **6** | 颜色 | 最弱，必须辅以其他变量 |

### 排版核心法则

| 法则 | 内容 |
|---|---|
| **变量叠加** | 标题 = 字号大 + 字重重 + 位置在上 + 留白多。 |
| **阈值跨越** | 层级差 ≥ 20%。 |
| **按需而动** | 三层级内容不需要五层级排版系统。 |
| **行宽呼吸** | 中文行宽25-45个汉字（约50-90ch，视字体而定）。行间距 ≥ 1.5（中文 ≥ 1.8）。 |

### 字号：约束空间中的诚实推演

| 约束 | 来源 |
|---|---|
| **阅读距离** | [PHYS] 碳基生理→正文16-18px |
| **人眼分辨率** | [PHYS] 碳基生理 |
| **韦伯层级阈值** | [PHYS] 碳基生理→层级差 ≥ 20% |
| **眼动对焦上限** | [PHYS] 碳基生理→跳跃 ≤ 33% |
| **字体渲染精度** | [ENG] 工程约定（4px网格，简化协作与心算） |
| **信息层级** | 内容逻辑 |

### 字号令牌

```css
:root {
  --font-size-caption:   0.75rem;  /* 12px | [PHYS] 平台最小可读字号下限 + [PHYS] 与body反向层级差≥20% + [ENG] 4px网格 | 帕累托最优解 */
  --font-size-body:      1rem;     /* 16px | [PHYS] 阅读距离+人眼分辨率 + [ENG] 浏览器默认基准 | 帕累托最优解之一，非唯一解 */
  --font-size-subtitle:  1.25rem;  /* 20px | [PHYS] +25%, [ENG] 4px网格 | 自由度: 18-20px，选20 */
  --font-size-h3:        1.5rem;   /* 24px | [PHYS] +20%, [ENG] 4px网格 | 自由度: 22-26px，选24 */
  --font-size-h2:        2rem;     /* 32px | [PHYS] +33%, [ENG] 4px网格 | 自由度: 30-34px，选32(空间结界) */
  --font-size-h1:        2.5rem;   /* 40px | [PHYS] +25%, [ENG] 4px网格 | 自由度: 36-42px，选40 */
}
```

h3→h2跳跃33%——h2开始承担空间分割功能。若内容层级感不强，可选`16→20→26→32→40`（h3为26px）。两者都在约束空间内。

**推演自**：3.2.1基石一（层级差异跨越韦伯阈值）、3.2.1基石四（格式塔相似性）、眼动跳跃生理、阅读距离与人眼分辨率、3.1.1渲染约束（4px工程约定）。

## 十 · 项目基因：Logo是项目的名字

**水之波光是血统。Logo是项目的名字。**

所有项目进去，一眼能认出是水之波光。Logo区分不同项目。不需要额外的环境纹理、偏色或角落标识——名字本身已是基因。

Logo位于结构层（导航栏或页眉），是"我在哪？"的直接回答。它属于内容层，不受涟漪层单色相约束。**品牌色由项目基因（[GENE]）决定，不在水之波光的物理约束链管辖范围内。** Logo在导航栏中的容器样式（背景、悬停状态、聚焦环）仍受十二条戒律约束。

**约束**：Logo是已有元素，零额外负债。最直接，最没有侵入感。用户看到它，潜意识确认——这是安心感。

## 十一 · 降级体系：约束链的动态重协商

**降级不是"减少功能"。降级是在约束链的薄弱环节诚实让步。**

当渲染环境从浏览器变为邮件客户端，JS约束消失——状态互斥自然重协商为CSS有限互斥。当设备性能受限，流转权最先让步，但反馈权不可撼动。水之波光不为任何技术时代而写。它绑定的是"诚实地跟随约束链演化"这一元规则。

**约束链重协商的两个时机**：

| 时机 | 说明 | 示例 |
|---|---|---|
| **设计时重协商** | 目标渲染环境在构建前已确定。降级是静态选择——直接输出适配该环境的代码。 | 邮件客户端→纯HTML。静态站点→无JS状态互斥。原生App→省略Web降级路径。 |
| **运行时重协商** | 目标环境在用户设备上动态变化。降级是动态响应——通过媒体查询、特性检测或性能监控实时调整。 | `prefers-reduced-motion`、低电量模式、OLED检测、FPS监测。 |

### 不可动摇的三层核心

| 层级 | 内容 |
|---|---|
| **第一层：哲学内核** | 三大实体、核心框架、约束链框架、推导纪律 |
| **第二层：碳基基石** | 感知约束与认知约束 |
| **第三层：工程底线** | 零硬编码、主权不容、内容唯一、因果不虚、碳基契约、状态逻辑互斥（JS环境中） |

### 可降级的四级阶梯

| 降级顺序 | 内容 | 降级方式 |
|---|---|---|
| **第一降级** | 引导流转 | 停止。交互流转保留（能耗可忽略）。纯黑模式下视觉流转不显现。 |
| **第二降级** | 涟漪调节（睫状肌微调节） | 停止。（v9.0.0 删除健康声明，此降级项保留为历史兼容） |
| **第三降级** | 动效细节 | 缓动降级为`ease-out`。动画降级或跳过。 |
| **第四降级** | 视觉质感 | 深度暗示改扁平。半透明改实色。遮罩退化为纯色。 |

### 不可降级的底线

| 底线 | 为何不可降级 |
|---|---|
| **反馈必须存在** | 因果不虚。 |
| **对比度 ≥ 4.5:1** | 碳基契约。 |
| **热区 ≥ 44px** | 碳基契约。 |
| **WCAG AA** | 包容万物。 |
| **设计令牌** | 零硬编码。 |
| **颜色双重编码** | 色觉障碍。 |
| **裁切双重指示** | 物理裁切+遮罩在目的需要时必须同时存在（极端密集场景经审后可降级）。 |
| **零态自证其空** | 虚空焦虑。纯黑模式下视觉压缩被接受或背景回退。 |
| **状态逻辑互斥** | JS环境中`data-state`机制不可降级。无JS环境中CSS有限互斥不是降级，是不同约束空间中的帕累托最优解。 |

### 降级裁决流程

```
1. 检测约束链变化：渲染环境？显示设备？设备性能？用户偏好？背景色模式？
2. 按序降级：从第一降级开始，逐级检查。满足条件即降级。
3. 触底停止：到达第三层即停止。宁可功能少，不可哲学破。
```

## 十三 · 等待之道

| 原则 | 含义 |
|---|---|
| **意义化** | 等待本身必须有意义。 |
| **让时间流动** | 骨架屏微呼吸持续至加载完成，或达到最大等待时间（如10秒）后转为确定性进度指示（如剩余时间估算）。绝不因"呼吸周期用完"而静止——静止暗示完成=不确定性=焦虑，违反情绪时间约束。进度条平滑推进。 |

**水之波光 v9.0.0 的诚实升级**：

> 纯黑不是敌人，也不是朋友。水蓝不是必然，也不是偶然。圆角不是进步，直角不是落后。微光不是健康，静默不是冷漠。
>
> 它们都是目的与约束挤压出的特定形态。v7.x-v8.0.0 的系统性错误在于：先有了"水之波光应该长什么样"的审美预设，然后用约束链为它颁发出生证明。v9.0.0 彻底删除这一预设。
>
> **水之波光没有默认形态。它只有默认的推导纪律。**
>
> 约束链不是单向管道，是目的驱动的加权网。韦伯阈值在"文本阅读"下权重极高，在"视频播放"下权重降低——但阈值本身从未消失。目的是权重分配器，不是豁免金牌。
>
> 四来源标注系统（[PHYS]/[ENG]/[GENE]/[PURPOSE]）确保每一条推导都可追溯、可审计、可质疑。工程约定不再冒充物理约束。项目基因不再写进约束链。审美惯性不再借哲学之名。
>
> 波光是相遇。触境是相知。微光是行为在水面的回响——但只在目的允许时。推演是约束空间中的诚实行走。约束链是相遇之前的全部物理现实——从验证过的目的开始，到认知结束。
>
> 框架的边界决定它能推导什么。升级框架的边界，就是升级哲学的边界。旧概念的诚实删除，就是保持约束链的干净。
