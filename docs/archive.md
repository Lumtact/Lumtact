## 4.0 项目目的档案

### 4.0.1 目的陈述

**目的**：
[Lumtract 用户] 在 [Web 浏览器/IDE 环境] 中，需要 [以可视化的方式理解复杂的 DAG 数据结构]，以达成 [快速理解系统拓扑与排查故障的价值]。

**最小信息集**：
1. 节点的层级与从属关系
2. 节点的唯一标识（ID/Label）
3. 节点之间的连接关系（边）
4. 节点的当前状态（正常/异常/处理中）

**高频路径**：
1. 页面加载 → 渲染初始图谱
2. Hover 节点 → 高亮相关节点与路径
3. Click 节点 → 展开详情面板

**零态场景**：
- 数据加载中：必须展示加载动画（Skeleton），禁止静止。
- 数据为空：必须有明确的"无数据"占位符。
- 数据加载失败：必须有明确的错误提示。

### 4.0.2 目的权重分配

| 约束类别 | 权重 | 理由 |
|---|---|---|
| 韦伯阈值（P-001） | 高 | 图谱节点密集，需要清晰的视觉层级区分 |
| 空间频率（P-002） | 中 | 节点边缘需配合抗锯齿，圆角提供辅助 |
| 工作记忆（P-020） | 中 | 用户需理解拓扑，但主要依赖空间记忆而非纯数字记忆 |
| 能耗预算（R-003） | 中 | 长时间监控工具，需避免闲置高能耗，但交互时需流畅 |

## 4.1 核心设计值推导记录

### 4.1.1 色彩系统

**背景色：#0f172a (Slate 900)**

```
来源标注：
  - [PHYS: D-006] 暗色模式降低暗环境眩光
  - [PHYS: P-001] 保留足够亮度空间供节点层级表达（差异 > 2% 可感知）
  - [GENE] Lumtract 深海发光品牌隐喻
  - [PURPOSE] 用户长时间注视，需减少视觉疲劳
```

**主强调色：#3b82f6 (Blue 500)**

```
来源标注：
  - [PHYS: P-003] 在深色背景上，蓝色提供最佳的对比度同时保持温和
  - [GENE] 科技感与冷静的工程基调
  - [PURPOSE] 强调交互状态与关键路径
```

### 4.1.2 形状与质感

**节点圆角：12px**

```
来源标注：
  - [PHYS: P-002] 圆角降低高频噪声，配合抗锯齿渲染
  - [ENG] 4px 网格倍数（3x），简化计算
  - [PURPOSE] 模拟"水滴"有机形态，软化工程结构的刚性
  - [GENE] Lumtract 有机美学基因
```

**容器玻璃态：bg-white/5 + backdrop-blur-md**

```
来源标注：
  - [PHYS: P-001] 极低透明度保留背景色调，维持层级深度
  - [PHYS: R-003] backdrop-blur 涉及 GPU 合成，仅在非密集元素上使用
  - [PURPOSE] 模拟水体密度感，增加界面深度

未选择纯色实心的理由：
  - 纯色实心会阻断背景光区流动，违反"波光是相遇"的公理

负债标记：无
```

## 4.2 冲突裁决记录

### 冲突 A：小屏幕下的节点点击热区

**冲突描述**：
在小屏幕（< 375px）上，节点可视化尺寸可能 < 20px（为了展示更多结构），但这违反了 **P-010**（热区最小 44px）。

**裁决顺位**：生存权（P-010） > 识读权（全览结构）

**裁决结果**：
- 保持热区 44px，即使视觉重叠。
- 在视觉上使用半透明描边扩大感知区域，但逻辑上判定点击半径扩大到 44px。

**来源标注**：
- [PHYS: P-010] 生存权强制要求，不可妥协。
- [PURPOSE] 移动端触摸操作准确性优先于全览的美观性。

**负债标记**：
- TODO: [DEBT] 小屏下节点可能视觉重叠 | 原因: 物理触控面积硬约束 | 演进: 引入手势缩放或列表降级视图

## 4.3 交互反馈推导

### 按钮：LumtractButton

**悬停状态**：
- 亮度提升至 `hover:bg-blue-500/20`
- 增加辉光 `hover:shadow-[0_0_20px_rgba(59,130,246,0.4)]`
- 转场时长 `duration-300`（在 P-005 舒适窗口内）

**推导理由**：
- [PHYS: P-003] 利用同时对比效应，深色背景上的微弱变化即可被感知。
- [PHYS: P-005] 300ms 落在注意力切换舒适区。
- [PHYS: P-019] 亮度变化频率极低，不会触发闪烁癫痫风险。
