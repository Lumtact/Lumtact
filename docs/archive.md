# 水之波光 · 触境
## The Water's Gleam · Tactile Realm
### 实例档案 v10.0.0
**性质**：应用记录。哲学、约束与推导引擎在特定项目中的具体实现轨迹。不是规范，是供审计的推导证据链。
---
## 4.0 实例档案概述
实例档案是**推导的证据**，不是**设计的规范**。它回答一个问题：**"这个设计值是怎么来的？"**
**档案与规范的关系**：
- 若档案中的推导无法被约束支撑 → 推导失败，设计值无效。
- 若档案中的流程无法被推导引擎解释 → 流程违规，推导无效。
- 若档案中的目的无法通过哲学的验证 → 目的无效，推导禁止启动。
**档案的不可变性**：档案一旦建立，只追加审计记录，不修改历史推导。错误推导通过新条目修正，旧条目保留为诚实履历。
---
## 4.1 目的档案（Purpose Profile）
每个项目必须建立独立的目的档案。
### 4.1.1 目的档案模板
```yaml
档案ID: [项目名]-purpose-[日期]
版本: v1.0.0
状态: 已验证 / 待验证 / 已修正
项目信息:
  项目名称: Lumtact
  服务实体: Lumtract Project
  项目类型: 数据可视化平台 / 设计系统 / ...
目的陈述:
  用户角色: [具体角色，如"开发者"、"内容消费者"、"系统管理员"]
  核心场景: [用户在什么物理/数字环境中使用？]
  核心任务: [用户需要完成的最高频任务是什么？]
  任务价值: [完成任务后用户获得什么？]
最小信息集:
  - [元素1: 为什么必须存在？删除后任务是否无法完成？]
  - [元素2: ...]
  - [元素N: ...]
高频行为序列:
  路径1:
    名称: [如"快速定位错误"]
    步骤:
      1. [行为1]
      2. [行为2]
      3. [行为3]
    频率: 每日 / 每小时 / 每次会话
零态场景分析:
  场景1:
    名称: [如"空文件夹"]
    出现频率: 高频 / 中频 / 低频
    上下文: [用户在此场景下的心理状态？是否焦虑？]
    自证需求: [视觉？触觉？听觉？]
目的验证记录:
  验证方式: 用户访谈 / 任务分析 / 行为数据 / 专家评估
  验证日期: [日期]
  验证结果: 通过 / 未通过 / 需修正
  验证人: [姓名/角色]
  备注: [任何需要说明的上下文]
目的权重分配:
  韦伯阈值(P-001): [极高/高/中/低] — 理由: [为什么在这个项目中是这个权重？]
  空间频率(P-002): [权重] — 理由: [...]
  工作记忆(P-020): [权重] — 理由: [...]
  能耗预算(R-003): [权重] — 理由: [...]
  环境融合(D-006): [权重] — 理由: [...]
  # 所有在推导中被激活的约束都必须有权重分配记录
```

### 4.1.2 目的档案示例

```yaml
档案ID: lumtact-purpose-2026-07-23
版本: v1.0.0
状态: 待验证
项目信息:
  项目名称: Lumtact
  服务实体: Lumtract Project
  项目类型: 数据可视化平台
目的陈述:
  用户角色: 系统架构师与开发者
  核心场景: 桌面Web环境，长时间注视
  核心任务: 理解复杂的DAG数据结构，排查故障节点
  任务价值: 在最小认知负荷下掌握系统拓扑
最小信息集:
  - 节点层级: 必须明确展示父子关系
  - 状态反馈: 操作后必须立即反馈状态变化
  - 路径追踪: 能够看到数据流向
高频行为序列:
  路径1:
    名称: 视图探索
    步骤:
      1. 打开页面
      2. 识别根节点
      3. 点击展开子节点
    频率: 每次会话
零态场景分析:
  场景1:
    名称: 无数据
    出现频率: 低频
    上下文: 刚刚加载，或查询无结果
    自证需求: 视觉（明确的空状态插画或文字）
目的权重分配:
  韦伯阈值(P-001): 极高 — 理由: 拓扑结构复杂，需清晰层级区分
  空间频率(P-002): 高 — 理由: 减少视觉噪点，提升长时间注视舒适度
  工作记忆(P-020): 中 — 理由: 图谱结构依赖外部记忆，而非纯工作记忆
  能耗预算(R-003): 中 — 理由: 仅为Web端，非移动端电池敏感场景
```

## 4.2 推导链记录（Derivation Chain Record）
每一个设计值必须有独立的推导链记录。
### 4.2.1 推导链记录模板

```yaml
记录ID: [项目名]-drv-[设计值名称]-[日期]
设计值:
  名称: [如 --color-bg-primary]
  值: [如 #121212]
  类型: 颜色 / 尺寸 / 时间 / 布尔 / 枚举
推导链:
  步骤1-目的激活:
    输入: [目的档案ID]
    输出: 激活约束 [约束ID列表]
    权重分配: [与目的档案一致]
  步骤2-约束挤压:
    硬边界排除:
      - 方案A: [值/方向] → 违反 [约束ID: 名称] → 排除
      - 方案B: [值/方向] → 违反 [约束ID: 名称] → 排除
    可行空间剩余:
      - 方向1: [描述]
      - 方向2: [描述]
  步骤3-帕累托最优:
    最优解集:
      - 解A: [值] — 优势: [...] — 劣势: [...]
      - 解B: [值] — 优势: [...] — 劣势: [...]
  步骤4-具体选择:
    选择: [最终值]
    选择理由: [为什么选这个而不是其他？]
    未选择记录:
      - [解X]: [理由]
来源标注:
  [PHYS]: 
    - [约束ID]: [具体约束如何支撑此值]
  [ENG]:
    - [约定名称]: [具体约定如何支撑此值]
  [GENE]:
    - [基因项]: [品牌/身份如何影响此值]
  [PURPOSE]:
    - [目的权重项]: [目的如何分配权重导致此值]
冲突记录:
  - 冲突ID: [如有]
  - 裁决结果: [如有]
负债标记:
  - TODO: [DEBT] [原因] [演进路径] / 无
审计状态:
  初审日期: [日期]
  初审结果: 通过 / 未通过
  审计人: [姓名]
```

### 4.2.2 推导链记录示例（背景色）

```yaml
记录ID: lumtact-drv-color-bg-primary-2026-07-23
设计值:
  名称: --color-bg-primary
  值: "#121212"
  类型: 颜色
推导链:
  步骤1-目的激活:
    输入: lumtact-purpose-2026-07-23
    输出: 激活约束 [P-001, P-002, D-004, D-006]
    权重分配: 韦伯阈值极高，空间频率高
  步骤2-约束挤压:
    硬边界排除:
      - 方案A: "#000000" → 违反 P-001（韦伯阈值）: 零态中频出现，纯黑压缩视觉层级空间，分割线/悬停差异低于1%阈值不可依赖 → 排除
      - 方案B: 浅色 "#FFFFFF" → 违反 [PURPOSE] 权重分配: 用户主要在暗室环境使用 → 权重低，排除
    可行空间剩余:
      - 方向1: 深灰 "#121212" — 保留足够亮度空间（~7%差异于纯黑），OLED节能
      - 方向2: 深灰 "#1A1A1A" — 更保守，但视觉沉闷
  步骤3-帕累托最优:
    最优解集:
      - 解A: "#121212" — 优势: OLED节能+层级空间充足+暗环境融合
  步骤4-具体选择:
    选择: "#121212"
    选择理由: 
      - 在OLED设备上同时满足D-004（节能）和P-001（层级空间）
      - 零态中频出现，需要视觉通道可靠自证
    未选择记录:
      - "#000000": 违反钻石约束P-001，不可妥协
来源标注:
  [PHYS]: 
    - D-004: OLED子像素关闭特性 → 降低能耗
    - P-001: 韦伯阈值 → 保留>2%亮度差异空间供层级表达
    - D-006: 环境光适应 → 暗环境降低眩光
  [GENE]:
    - Lumtact 深海隐喻: 品牌身份表达（已验证不违反钻石约束）
  [PURPOSE]:
    - 复杂拓扑任务: 零态中频，需完整视觉层级
冲突记录: 无
负债标记: 无
```
