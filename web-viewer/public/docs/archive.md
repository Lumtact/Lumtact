## 4.0 项目目的档案

### 目的陈述

**Lumtract 用户** 在 **Web 浏览器** 中，需要 **以可视化的方式理解复杂的 DAG 数据结构**，以达成 **快速理解系统拓扑与排查故障** 的价值。

**最小信息集**：
1. 节点的层级与从属关系
2. 节点的唯一标识（ID/Label）
3. 节点之间的连接关系（边）
4. 节点的当前状态（正常/异常/处理中）

**高频路径**：
1. 页面加载 → 渲染初始图谱
2. Hover 节点 → 高亮相关节点与路径
3. Click 节点 → 展开详情面板

**零态场景**：
- 数据加载中 → Skeleton 动画（禁止静止）
- 数据为空 → 明确占位符
- 数据加载失败 → 明确错误提示

### 目的权重分配（关键约束）

| 约束 | 权重 | 理由 |
|---|---|---|
| P-001 韦伯阈值 | 高 | 密集节点需清晰层级 |
| P-002 空间频率 | 高 | 减少视觉噪点，提升长时间注视舒适度 |
| P-010 热区 ≥ 44px | 高 | 触控设备使用 |
| P-020 工作记忆 | 中 | 依赖空间记忆 > 数字记忆 |
| R-003 能耗 | 中 | 长时间使用，闲置时停止动画 |
| D-006 环境融合 | 中 | 暗室场景需暗色模式 |

**目的验证状态**：待验证（需与真实用户对齐）

## 4.1 关键推导链记录

### 4.1.1 背景色：`#0f172a` (Slate 900)

**推导链**：

1. **目的激活**：
   - 输入: Lumtract 目的档案 (v1.0.0)
   - 激活约束: P-001, P-002, D-004, D-006
   - 权重分配: 韦伯阈值极高，空间频率高

2. **硬边界排除**：
   - `#000000` → 违反 P-001（韦伯阈值）：零态中频出现，纯黑压缩视觉层级空间，分割线/悬停差异低于1%阈值不可依赖 → 排除
   - `#F8F9FA`（浅色）→ 不违反钻石，但与 [PURPOSE] 暗色偏好冲突 → 纳入可行空间但权重降低
   - `#0A0A0A` → 违反 P-001：仅比纯黑略好，仍不足 → 排除

3. **可行空间剩余**：
   - 方向1: `#0f172a` (Slate 900) — 保留约7%亮度差异于纯黑，OLED节能，暗环境融合
   - 方向2: `#121212` — 更节能 (OLED)，但层级空间略小
   - 方向3: `#F8F9FA` — 日光场景优，暗室眩光

4. **帕累托最优**：以上三者在不同目的下均为最优解。

5. **具体选择**：`#0f172a`
   - [PURPOSE] 暗室监控子场景权重高
   - [GENE] Lumtract 深海发光品牌隐喻
   - [PHYS: D-004] OLED 节能（虽不如 #121212，但可接受）
   - [PHYS: P-001] 保留约 7% 亮度差异空间

**来源标注**：
- [PHYS: D-004] OLED 子像素关闭特性 → 降低能耗
- [PHYS: P-001] 韦伯阈值 → 保留 >2% 亮度差异空间供层级表达
- [PHYS: D-006] 环境光适应 → 暗环境降低眩光
- [GENE] Lumtract 深海发光隐喻 → 品牌身份表达（已验证不违反钻石约束）
- [PURPOSE] 复杂拓扑任务 → 零态中频，需完整视觉层级

**负债标记**：
- TODO: [DEBT] 日光场景下 `#0f172a` 可能眩光 | 演进: 运行时环境光检测，自动切换浅色主题

### 4.1.2 主强调色：`#3b82f6` (Blue 500)

**推导链**：

1. **目的激活**：激活约束 P-003（同时对比效应）
2. **硬边界排除**：未违反任何钻石/钢铁约束。
3. **可行空间**：多种蓝色、青色、紫色均在 sRGB 安全范围内。
4. **帕累托最优**：不同蓝色在色相、饱和度、明度上各有优劣。
5. **具体选择**：`#3b82f6`
   - [GENE] 科技感 + 冷静基调（与 Lumtract 品牌一致）
   - [PHYS: P-003] 同时对比效应：深色背景上蓝色温和且显著
   - [PURPOSE] 强调交互状态，非装饰性色彩

**来源标注**：
- [PHYS: P-003] 同时对比效应 → 深色背景上蓝色感知显著
- [GENE] Lumtract 品牌调性 → 科技感与冷静
- [PURPOSE] 交互强调 → 非装饰性色彩

**负债标记**：无

---

### 4.1.3 节点圆角：`12px`

**推导链**：

1. **目的激活**：激活约束 P-002（空间频率）、[ENG] 4px 网格
2. **硬边界排除**：无钻石/钢铁约束强制特定圆角值。
3. **可行空间**：4px、8px、12px、16px 均为合法选择（受 [ENG] 4px 网格约束）。
4. **帕累托最优**：
   - 8px（更工程、更硬朗）
   - 12px（更有机、更柔和）
   - 16px（更温和、更圆润）
5. **具体选择**：12px
   - [PURPOSE] "水滴"有机形态，软化工程刚性
   - [ENG] 4px × 3 = 12px，网格对齐
   - [PHYS: P-002] 空间频率：圆角降低高频噪声

**来源标注**：
- [PHYS: P-002] 空间频率 → 圆角降低高频噪声，配合抗锯齿渲染
- [ENG] 4px 网格 → 3x 倍数，简化计算
- [GENE] Lumtract 有机美学 → 模拟"水滴"形态
- [PURPOSE] 复杂拓扑界面 → 软化边界，提升长时间注视舒适度

**负债标记**：无

## 4.2 冲突裁决记录

### 冲突 A：小屏幕热区 vs 图谱全览

| 项目 | 内容 |
|---|---|
| **约束 A** | P-010 热区 ≥ 44px（钻石） |
| **约束 B** | 识读权：全览图谱结构（识读权） |
| **冲突场景** | 小屏幕 (<375px) 下，节点视觉尺寸可能 <20px |
| **裁决** | 生存权 > 识读权 → P-010 胜出 |
| **最终方案** | 热区保持 44px（逻辑判定），视觉上使用半透明描边扩大感知区域 |
| **来源标注** | [PHYS: P-010] 生存权强制要求；[PURPOSE] 移动端触摸准确性优先 |
| **负债** | TODO: [DEBT] 小屏节点视觉重叠 | 演进: 引入手势缩放 |

---

### 冲突 B：品牌暗色 vs 日光场景浅色

| 项目 | 内容 |
|---|---|
| **约束 A** | [GENE] Lumtract 暗色品牌偏好（陶土） |
| **约束 B** | [PURPOSE] 日光场景浅色更优（目的权重） |
| **冲突场景** | 同一界面需适应暗室和日光两种环境 |
| **裁决** | 暗室为高频子场景，浅色可通过运行时主题切换覆盖 |
| **最终方案** | 默认深色，后续实现主题切换 |
| **来源标注** | [PURPOSE] 暗室场景权重高于日光；[ENG] 主题切换为工程可行方案 |
| **负债** | TODO: [DEBT] 运行时主题切换 | 演进: 检测环境光自动切换 |
